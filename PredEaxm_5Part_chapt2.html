벤 프라이의 7단계 방법론은 정보 획득(Acquire), 분해(Parse), 선별(Filter), 마이닝(Mine), 표현(Represent), 정제(Refine), 상호작용(Interact)이다.
비기봇 해설
벤 프라이(Ben Fry)는 데이터 시각화를 위한 7단계 방법론을 제안했는데, 이 방법론은 다음과 같은 단계로 구성됩니다:

1. Acquire (획득): 데이터를 수집하거나 접근할 수 있는 방법을 마련합니다.
2. Parse (파싱): 수집된 데이터를 분석하고 구조화하여 사용할 수 있는 형식으로 변환합니다.
3. Filter (필터링): 데이터에서 필요한 부분만을 추출하여 불필요한 데이터를 제거합니다.
4. Mine (채굴): 데이터를 탐색하여 패턴이나 의미 있는 정보를 발견합니다.
5. Represent (표현): 데이터를 적절한 시각적 형태로 표현할 방법을 선택합니다.
6. Refine (정제): 표현된 데이터를 더 명확하고, 이해하기 쉽게 다듬습니다.
7. Interact (상호작용): 사용자가 데이터와 상호작용할 수 있도록 인터페이스를 설계합니다.

따라서 3번이 부적절합니다.

LATCH 방법에서의 시간은 Time이며, 여기서 H는 위계(Hierarchy)이다.
비기봇 해설
정답은 4번입니다.

LATCH 방법은 정보의 조직화를 위한 다섯 가지 기준을 제시합니다. 이 방법은 정보의 구조화와 정리를 돕기 위해 사용되며, 각각의 기준은 특정한 방식으로 정보를 배열합니다. LATCH는 위치(Location), 알파벳(Alphabet), 시간(Time), 카테고리(Category), 위계(Hierarchy)로 구성됩니다. 이 기준들은 정보를 효율적으로 정리하고, 사용자가 필요로 하는 정보를 쉽게 찾을 수 있도록 돕습니다.

1. 위치(Location)
: 위치(Location)는 정보를 공간적인 기준으로 배열하는 방법입니다. 이는 지리적 위치뿐만 아니라, 물리적 또는 가상 공간에서의 위치를 기준으로 정보를 정리합니다. 예를 들어, 나라별로 정보를 분류하거나, 특정 장소에 기반한 데이터를 정리할 때 사용됩니다. LATCH의 기준 중 하나로, 정보의 물리적 위치를 통해 쉽게 접근할 수 있도록 합니다.

2. 알파벳(Alphabet)
: 알파벳(Alphabet)은 정보를 알파벳 순서로 배열하는 방법입니다. 이는 사전식 배열을 통해 정보를 정리하며, 이름이나 제목과 같은 텍스트 기반의 정보를 정리할 때 유용합니다. LATCH의 기준 중 하나로, 사용자가 특정 이름이나 단어를 쉽게 찾을 수 있도록 돕습니다.

3. 카테고리(Category)
: 카테고리(Category)는 정보를 특정 그룹이나 유형으로 분류하는 방법입니다. 이는 정보를 주제나 성격에 따라 그룹화하여 정리합니다. 예를 들어, 도서관에서 책을 장르별로 분류하는 것과 같습니다. LATCH의 기준 중 하나로, 관련된 정보를 함께 묶어 사용자가 쉽게 탐색할 수 있도록 합니다.

4. 시간(Hour)
: 시간(Hour)은 정보를 시간의 흐름에 따라 배열하는 방법입니다. 이는 사건이나 데이터를 발생한 시간 순서대로 정리하여, 시간의 경과에 따른 변화를 쉽게 파악할 수 있도록 합니다. 그러나 LATCH의 시간 기준은 'Time'으로, 'Hour'는 LATCH의 기준에 포함되지 않습니다. 따라서 이 선지는 LATCH 방법에 해당하지 않습니다.

LATCH 방법의 다섯 가지 기준은 위치(Location), 알파벳(Alphabet), 시간(Time), 카테고리(Category), 위계(Hierarchy)입니다. 선지 4의 '시간(Hour)'는 LATCH의 시간 기준인 'Time'과 다르며, LATCH 방법에 포함되지 않습니다. 따라서 정답은 4번입니다. 나머지 선지들은 모두 LATCH의 기준에 해당합니다.
따라서, 정답은 '4. 시간(Hour)' 입니다.
