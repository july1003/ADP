해설
B2B기업은 기업 간의 전자 상거래를 진행하여 지속적인 데이터가 생성 되기 어렵다. 반면에 B2C기업은 고객을 상대로 하기 때문에 고객의 데이터가 지속적으로 생성 된다.
비기봇 해설
정답은 3번입니다.

빅데이터 시대에는 데이터를 많이 확보할 수 있는 기업이 혁신과 경쟁력 향상에 유리합니다. 데이터의 양과 질은 기업의 경쟁력을 좌우하는 중요한 요소입니다.

1. 신용카드회사
: 신용카드회사는 고객의 거래 데이터를 통해 소비 패턴을 분석하고, 이를 기반으로 다양한 금융 상품을 개발할 수 있습니다.

2. 여행회사
: 여행회사는 고객의 여행 패턴과 선호도를 분석하여 맞춤형 여행 상품을 제공할 수 있지만, 데이터의 양과 질이 다른 업종에 비해 상대적으로 적을 수 있습니다.

3. B2B기업
: B2B기업은 주로 기업 간 거래 데이터를 다루며, 데이터의 양이 많지 않을 수 있습니다. 또한, 데이터의 활용 범위가 제한적일 수 있습니다.

4. 이동통신사
: 이동통신사는 고객의 통화 및 데이터 사용 패턴을 통해 다양한 분석을 수행할 수 있으며, 방대한 양의 데이터를 보유하고 있습니다.


B2B기업은 다른 선택지에 비해 데이터를 확보할 수 있는 기회가 상대적으로 적습니다. 신용카드회사와 이동통신사는 방대한 양의 고객 데이터를 보유하고 있으며, 여행회사는 고객의 여행 데이터를 통해 맞춤형 서비스를 제공할 수 있습니다. 따라서 B2B기업이 빅데이터 시대에 부합되기 어려운 기업 분류입니다.

따라서, 정답은 '3. B2B기업' 입니다.

10. SQL 명령은 크게 DML과 DDL로 분류된다. 다음 SQL 명령 중 그 성격이 나머지와 다른 것은?
  1. CREATE
  2. DELETE
  3. SELECT
  4. INSERT
정답보기
정답
1 CREATE

해설
DML(Data Manipulation Language)는 테이블에 데이터 검색, 삽입, 수정, 삭제하는데 사용하며 SELECT, INSERT, UPDATE, DELETE 등이 있다. DDL(Data Definition Language)는 테이블이나 관계의 구조를 생성하는데 사용하며 CREAT, DROP, ALTER, TRUNCATE 등이 있다.
비기봇 해설
정답은 1번입니다.

SQL 명령어는 데이터베이스를 조작하는 데 사용되며, 크게 DML(Data Manipulation Language)과 DDL(Data Definition Language)로 나뉩니다. DML은 데이터의 삽입, 삭제, 수정, 조회를 담당하고, DDL은 데이터베이스 구조의 정의와 관리를 담당합니다.

1. CREATE
: 데이터베이스의 테이블이나 인덱스와 같은 구조를 정의하거나 수정하는 데 사용됩니다. 이는 데이터의 구조를 설정하는 명령어로, 데이터 자체를 조작하지 않습니다.

2. DELETE
: 데이터베이스에서 특정 데이터를 삭제하는 데 사용됩니다. 이는 데이터 자체를 조작하는 명령어로, 데이터의 구조에는 영향을 미치지 않습니다.

3. SELECT
: 데이터베이스에서 데이터를 조회하는 데 사용됩니다. 이는 데이터 자체를 조작하는 명령어로, 데이터의 구조에는 영향을 미치지 않습니다.

4. INSERT
: 데이터베이스에 새로운 데이터를 삽입하는 데 사용됩니다. 이는 데이터 자체를 조작하는 명령어로, 데이터의 구조에는 영향을 미치지 않습니다.


CREATE 명령어는 데이터베이스의 구조를 정의하는 DDL에 속하며, 나머지 명령어들은 데이터 자체를 조작하는 DML에 속합니다. 따라서 CREATE가 다른 성격을 가집니다.

따라서, 정답은 '1. CREATE' 입니다.

데이터웨어하우스는 비휘발성(Non Volatile)을 특징을 가지고 있으므로, 일단 데이터가 적재되면 일괄 처리(Batch 처리) 외에는 별도의 갱신이 이루어지지 않는다. 데이터웨어하우스는 데이터를 주제별로 저장하기 때문에 최종사용자 혹은 전산 지식에 약한 분석자도 이해하기 쉬운 형태를 취한다.
비기봇 해설
정답은 3번입니다.

데이터 웨어하우스는 대량의 데이터를 저장하고 분석하기 위한 시스템으로, 주로 의사결정 지원을 목적으로 사용됩니다. 이는 운영 데이터베이스와는 다른 특성을 가지며, 주로 데이터 분석과 보고에 중점을 둡니다.

1. 데이터에 대한 수정 및 갱신이 자주 발생한다.
: 데이터 웨어하우스는 주로 데이터의 저장과 분석을 목적으로 하며, 데이터의 수정 및 갱신이 빈번하게 발생하지 않습니다. 이는 운영 데이터베이스와의 주요 차이점 중 하나입니다.

2. 거래(Transaction)를 처리하는데 주로 이용된다.
: 거래 처리는 주로 운영 데이터베이스에서 이루어지며, 데이터 웨어하우스는 대량의 데이터를 분석하고 보고하는 데 중점을 둡니다. 따라서 거래 처리와는 목적이 다릅니다.

3. 데이터웨어하우스 내부의 데이터는 최종 사용자(end user)나 전산에 약한 분석가도 이해하기 쉬운 형태이다.
: 데이터 웨어하우스는 최종 사용자나 분석가가 쉽게 이해할 수 있도록 데이터를 구조화하고, 다양한 분석 도구를 통해 접근할 수 있도록 설계됩니다. 이는 데이터 웨어하우스의 주요 장점 중 하나입니다.

4. 입출력보다는 연산 작업이 중심이 된다.
: 데이터 웨어하우스는 대량의 데이터를 저장하고 분석하는 데 중점을 두며, 입출력보다는 데이터의 집계와 분석 작업이 중심이 됩니다.


데이터 웨어하우스는 최종 사용자나 분석가가 쉽게 이해할 수 있도록 데이터를 구조화하여 제공하는 것이 주요 목적입니다. 다른 선택지들은 데이터 웨어하우스의 특성과 맞지 않으며, 주로 운영 데이터베이스의 특성에 가깝습니다.

  1. 1개
  2. 2개
  3. 3개
  4. 4개
정답보기
정답
3 3개

해설
맵리듀스에서 블록 크기의 기본값은 64MB이고, 맵 태스크 하나가 1개의 블록을 대상으로 연산을 수행한다. 따라서 320MB 크기의 파일을 대상으로 작업을 수행하면 5개의 맵 태스크가 생성된다. (320MB / 64MB = 5개)
비기봇 해설
MapReduce에서 별도의 옵션을 지정하지 않고 작업을 수행할 때, 기본적으로 HDFS 블록 크기에 따라 맵 태스크(Map Task)의 개수가 결정됩니다. 기본 블록 크기는 보통 128MB입니다. 따라서 320MB 크기의 파일이 몇 개의 블록으로 나뉘는지 계산해 봅니다.

320MB 파일을 128MB 블록 크기로 나누면
320/128≈2.5

따라서, 320MB 파일을 128MB 블록 크기로 나누었을 때 정확하게 생성되는 맵 태스크의 개수는 3개입니다.

2. 데이터 스테이징 단계에서는 일괄(Batch) 작업 형태의 실시간 ETL과 정기적인 ETL을 혼용할 수 없다.
: 데이터 스테이징 단계에서는 정기적인 ETL과 실시간 ETL을 혼용할 수 있습니다. 이는 다양한 데이터 처리 요구사항을 충족시키기 위해 필요합니다. 예를 들어, 일부 데이터는 실시간으로 처리되어야 할 수 있으며, 다른 데이터는 정기적으로 일괄 처리될 수 있습니다. 따라서 두 가지 방식의 ETL을 혼용하여 유연한 데이터 처리 환경을 구축할 수 있습니다.

14. 다음 중 하둡 분산파일시스템(HDFS)의 특징에 대한 설명으로 가장 적절하지 않은 것은?

  3. HDFS는 데이터를 파일 단위로 저장하는 시스템이다.
  4. HDFS에서 기본적으로 파일은 한 번 쓰이면 변경되지 않는다고 가정한다.

HDFS는 파일 데이터를 블록이나 청크 단위로 저장하며, 데이터에 대한 순차 접근 방식을 지원한다.
비기봇 해설
정답은 3번입니다.

하둡 분산파일시스템(HDFS)은 대용량 데이터를 효율적으로 저장하고 처리하기 위해 설계된 시스템입니다. 주로 대규모 데이터 세트를 처리하는 데 사용되며, 데이터의 안정성과 가용성을 높이기 위해 여러 노드에 데이터를 복제하여 저장합니다.

1. HDFS는 데이터의 랜덤 접근 방식을 지원하지 않는다.
: HDFS는 대용량 데이터의 순차적 접근을 최적화하기 위해 설계되었습니다. 따라서 랜덤 접근보다는 대량의 데이터를 한 번에 읽는 작업에 더 적합합니다.

2. HDFS는 구글 파일 시스템의 아키텍처와 사상을 그대로 구현한 클로닝(Cloning) 프로젝트이다.
: HDFS는 구글 파일 시스템(GFS)의 아키텍처를 기반으로 개발되었습니다. GFS의 설계 철학을 따르며, 대규모 데이터 처리에 적합한 구조를 가지고 있습니다.

3. HDFS는 데이터를 파일 단위로 저장하는 시스템이다.

2. Data Complexity
: 데이터의 복잡성은 데이터의 구조와 형태가 얼마나 복잡한지를 나타냅니다. 복잡한 데이터는 다양한 형식과 구조를 가질 수 있으며, 이를 분석하기 위해서는 데이터 전처리와 변환 과정이 필요합니다. 예를 들어, 비정형 데이터나 다차원 데이터는 복잡성이 높아 이를 처리하기 위한 특별한 기술이 요구됩니다. 데이터 복잡성은 분석의 난이도를 결정짓는 중요한 요소입니다.

3. Speed

분석과제의 주요 관리 영역은 데이터의 크기, 복잡성, 그리고 처리 속도와 같은 요소들입니다. 이러한 요소들은 분석의 효율성과 효과성을 직접적으로 결정짓는 반면, '분석 및 접근성'은 분석 결과의 활용 측면에 더 가깝습니다.

따라서, 정답은 '4. Analytic & Accessibility' 입니다.

: 기하분포는 이산형 확률분포의 한 종류로, 특정 사건이 처음으로 발생할 때까지의 시행 횟수를 모델링합니다. 예를 들어, 동전을 던져 처음으로 앞면이 나올 때까지의 횟수를 구하는 경우가 이에 해당합니다. 기하분포는 성공 확률이 p인 베르누이 시행에서 처음 성공할 때까지의 시행 횟수를 나타내며, 확률질량함수(PMF)를 통해 각 시행 횟수에 대한 확률을 계산할 수 있습니다.

2. 이항분포
: 이항분포는 이산형 확률분포로, n번의 독립적인 베르누이 시행에서 성공 횟수를 모델링합니다. 예를 들어, 10번의 동전 던지기에서 앞면이 나오는 횟수를 구하는 경우가 이에 해당합니다. 이항분포는 성공 확률이 p인 베르누이 시행을 n번 반복했을 때, k번 성공할 확률을 계산하는 데 사용되며, 이항계수를 활용하여 확률을 구합니다.

3. 지수분포
: 지수분포는 연속형 확률분포로, 사건이 발생할 때까지의 시간 간격을 모델링합니다. 예를 들어, 고객이 매장에 도착할 때까지의 시간을 구하는 경우가 이에 해당합니다. 지수분포는 연속적인 시간 간격을 다루며, 확률밀도함수(PDF)를 통해 특정 시간 구간 내에서 사건이 발생할 확률을 계산합니다. 따라서, 지수분포는 이산형 확률분포가 아닙니다.

4. 초기하분포
: 초기하분포는 이산형 확률분포로, 유한 모집단에서 비복원 추출을 통해 특정 속성을 가진 항목의 개수를 모델링합니다. 예를 들어, 10개의 공 중에서 3개의 빨간 공을 뽑는 경우가 이에 해당합니다. 초기하분포는 모집단에서의 성공과 실패의 수를 고려하여, 비복원 추출 시 특정 개수의 성공을 얻을 확률을 계산합니다.

이산형 확률분포는 확률변수가 특정한 개수의 값을 가질 수 있는 경우를 말합니다. 기하분포, 이항분포, 초기하분포는 모두 이산형 확률분포로, 각각 특정 사건이 발생할 때까지의 시행 횟수, 성공 횟수, 비복원 추출에서의 성공 개수를 모델링합니다. 반면, 지수분포는 연속형 확률분포로, 사건이 발생할 때까지의 시간 간격을 모델링하므로 이산형 확률분포에 해당하지 않습니다.

3. 원 자료로 추정한 계수가 가장 큰 변수

4. 민감도
: 민감도는 실제로 질병이 있는 사람 중에서 검사 결과가 양성으로 나올 확률을 의미하는 지표로, 특이도와 마찬가지로 주로 의료 분야에서 사용됩니다. 연관분석의 지표와는 관련이 없으며, 문제에서 요구하는 연관성의 정도를 나타내는 데 적합하지 않습니다.

해설
정규성 가정을 충족하지 못해 데이터를 추가 수집하더라도 데이터 수집 과정에서 이미 발생한 오차나 불확실성이 모델에 영향을 미칠 가능성이 있기 때문이다.
비기봇 해설
정답은 4번입니다.

잔차분석에서 정규성 검토는 회귀 분석의 가정 중 하나로, 잔차가 정규분포를 따르는지를 확인하는 과정입니다. 이는 회귀 모델의 적합성을 평가하는 데 중요한 역할을 합니다.

1. 정규확률그림(Q-Q Plot)은 정규성을 평가하는 하나의 방편이지 절대적인 기준은 아니다.
: Q-Q Plot은 잔차의 정규성을 시각적으로 평가하는 도구입니다. 잔차가 대각선에 가깝게 분포하면 정규성을 만족한다고 판단할 수 있지만, 이는 절대적인 기준이 아닙니다. Q-Q Plot은 정규성 검토의 보조적인 수단으로 사용되며, 다른 정규성 검정과 함께 사용하여 보다 정확한 판단을 내리는 것이 중요합니다.

2. 잔차의 히스토그램이나 점도표를 그려서 정규성 문제를 검토하기도 한다.
: 잔차의 히스토그램이나 점도표는 잔차의 분포를 시각적으로 확인하는 방법입니다. 이러한 시각적 도구들은 잔차가 정규분포를 따르는지에 대한 초기 판단을 내리는 데 유용합니다. 그러나 시각적 검토만으로는 충분하지 않으며, 통계적 검정을 통해 정규성을 확인하는 것이 필요합니다.

3. 정규성을 검정하는 방법으로 Shapiro-Wilk Test, Anderson-Darling Test 등을 이용할 수 있다.
: Shapiro-Wilk Test와 Anderson-Darling Test는 잔차의 정규성을 검정하는 통계적 방법입니다. 이 검정들은 귀무가설로 '데이터가 정규분포를 따른다'를 설정하고, p-value를 통해 정규성을 판단합니다. p-value가 유의수준보다 크면 정규성을 만족한다고 볼 수 있습니다.

4. 정규성 가정을 충족하지 못한 데이터 추가 수집 등을 통해 해결할 수 있다.
: 정규성 가정을 충족하지 못한 경우, 데이터 변환이나 추가 데이터 수집을 통해 문제를 해결할 수 있습니다. 그러나 단순히 데이터를 추가 수집하는 것만으로 정규성을 보장할 수는 없습니다. 데이터 변환이나 다른 통계적 방법을 통해 정규성을 확보하는 것이 중요합니다. 따라서 이 선지는 부적절한 설명입니다.

선지 4는 정규성 가정을 충족하지 못한 경우의 해결책으로 '데이터 추가 수집'을 제시하고 있습니다. 그러나 데이터 추가 수집만으로 정규성을 보장할 수는 없으며, 데이터 변환이나 다른 통계적 방법을 통해 정규성을 확보하는 것이 중요합니다. 따라서 이 선지는 부적절한 설명입니다. 나머지 선지들은 정규성 검토 방법에 대한 적절한 설명을 제공하고 있습니다.

2. 자율적인(Unsupervised) 신경망 모델로서 역전파(Back Propagation) 알고리즘처럼 여러 단계의 피드백을 처리하면서 전방 패스(Feed-Forward Flow)를 사용하는 방법이다.

4. 고차원의 표현을 1차원으로 표현할 수 있는 장점이 있다.

선지 2는 SOM의 학습 방식에 대한 잘못된 설명을 포함하고 있습니다. SOM은 비지도 학습 알고리즘으로, 역전파 알고리즘과 같은 피드백 과정을 사용하지 않습니다. 대신, 단일 전방 패스를 통해 입력 벡터와 가장 유사한 뉴런을 찾고, 그 뉴런과 주변 뉴런의 가중치를 조정하는 방식으로 학습합니다. 따라서, 선지 2는 SOM의 특성을 잘못 설명하고 있어 정답입니다. 다른 선지들은 SOM의 특성을 올바르게 설명하고 있습니다.

1. 연결정도 중심성(Degree centrality)
: 연결정도 중심성은 네트워크 내에서 특정 노드가 얼마나 많은 다른 노드와 직접 연결되어 있는지를 측정하는 지표입니다. 이는 네트워크에서의 영향력이나 중요성을 평가하는 데 사용되며, 연결된 노드의 수가 많을수록 중심성이 높아집니다. 따라서 연결정도 중심성은 네트워크 구조를 파악하는 데 유용한 기법입니다.

2. 근접 중심성(Closeness centrality)
: 근접 중심성은 특정 노드가 네트워크 내의 다른 모든 노드에 얼마나 빠르게 도달할 수 있는지를 측정합니다. 이는 네트워크의 중앙에 위치한 노드를 식별하는 데 사용되며, 근접 중심성이 높을수록 네트워크의 중심에 위치하게 됩니다. 따라서 근접 중심성은 네트워크 구조를 이해하는 데 중요한 역할을 합니다.

3. 위세 중심성(Eigenvactor centrality)
: 위세 중심성은 노드의 연결정도뿐만 아니라 연결된 노드들의 중요성까지 고려하여 노드의 중요성을 평가합니다. 이는 네트워크 내에서 영향력 있는 노드와의 연결이 많을수록 자신의 위세도 높아지는 원리를 기반으로 합니다. 위세 중심성은 네트워크 구조를 파악하는 데 필수적인 기법입니다.

: 노드 중심성이라는 용어는 일반적으로 사용되지 않으며, 네트워크 분석에서 특정한 중심성 지표로 인정되지 않습니다. 이는 문제에서 제시된 다른 중심성 지표들과 달리, 네트워크 구조를 파악하기 위한 기법으로 사용되지 않기 때문에 오답입니다.
