(c) 메가바이트(Megabyte)
: 1MB = 약 100만 바이트(정확히는 1,048,576 Byte). 비교 기준 중 가장 작은 단위입니다.

(a) 기가바이트(Gigabyte)

(b) 테라바이트(Terabyte)

SQL on Hadoop은 이러한 Hadoop의 기능을 SQL과 결합하여 사용자가 익숙한 SQL 쿼리를 통해 대량의 데이터를 처리할 수 있도록 합니다. 이를 통해 데이터 분석가나 개발자가 대량의 데이터를 보다 쉽게 다룰 수 있게 됩니다.

: Hive는 Hadoop 위에서 SQL과 유사한 쿼리를 실행할 수 있도록 해주는 데이터 웨어하우스 시스템입니다. Hive는 대량의 데이터를 쿼리하고 분석하는 데 최적화되어 있으며, SQL과 유사한 HiveQL을 사용하여 데이터를 처리합니다. 따라서 Hive는 SQL on Hadoop의 구성 요소로 적합합니다.

2. Pig
: Pig는 대량의 데이터를 분석하기 위한 플랫폼으로, Pig Latin이라는 스크립트 언어를 사용하여 데이터를 처리합니다. Pig는 복잡한 데이터 변환 작업을 수행하는 데 유용하며, Hadoop과 함께 사용되어 대량의 데이터를 처리할 수 있습니다. 따라서 Pig도 SQL on Hadoop의 구성 요소로 적합합니다.

3. Spark
: Spark는 대규모 데이터 처리를 위한 클러스터 컴퓨팅 프레임워크로, Hadoop과 함께 사용될 수 있습니다. Spark는 메모리 내에서 데이터를 처리하여 빠른 속도를 제공하며, SQL과 유사한 Spark SQL을 통해 데이터를 쿼리할 수 있습니다. 따라서 Spark는 SQL on Hadoop의 구성 요소로 적합합니다.
