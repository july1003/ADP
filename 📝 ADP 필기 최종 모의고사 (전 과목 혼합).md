## Ⅰ. 데이터 이해

### 문제 1. 데이터 품질 관리의 중요성

### ✅ 모범답안

> 데이터 품질은 분석 결과의 신뢰성과 직결되며, 품질이 낮을 경우 잘못된 의사결정을 초래할 수 있다.
> 예를 들어 결측치나 오류 데이터가 많을 경우 모델 성능이 저하되거나 왜곡된 분석 결과가 도출될 수 있다.
> 따라서 정확성, 완전성, 일관성 등의 데이터 품질 관리가 필수적이다.

### 📌 채점 기준

- “신뢰성” 언급 ✔
- 문제점 구체화 ✔
- 품질 요소 언급 ✔
  → **만점 답안**

------

### 문제 2. 정형 데이터 vs 비정형 데이터

### ✅ 모범답안

> 정형 데이터는 고정된 스키마를 가진 데이터로 관계형 데이터베이스에 저장된다.
> 비정형 데이터는 스키마가 없거나 불규칙한 형태로 텍스트나 이미지 등이 이에 해당한다.
> 비정형 데이터 분석 시에는 전처리와 특징 추출 과정이 중요하다.

### 문제 4. KPI를 명확히 정의해야 하는 이유

### ✅ 모범답안

> KPI는 분석 결과가 목표 달성에 기여하는지를 판단하는 기준이 된다.
> KPI가 불명확할 경우 분석 결과의 성공 여부를 평가할 수 없으며, 실무 활용이 어려워진다.

------

### 문제 5. CRISP-DM 활용 이유

### ✅ 모범답안

> CRISP-DM은 비즈니스 이해를 출발점으로 하여 분석 결과의 배포까지 전 과정을 포괄하는 방법론이다.
> 반복 가능하고 실무 적용이 용이하여 데이터 분석 프로젝트에서 널리 활용된다.

> 과적합은 학습 데이터에서는 성능이 높으나 새로운 데이터에 대한 일반화 성능이 낮은 상태이다.

### 문제 11. Random Forest 우수성

### ✅ 모범답안

> Random Forest는 서로 다른 데이터 샘플과 변수로 학습된 다수의 트리를 결합한다.
> 이를 통해 분산을 감소시켜 과적합을 완화하고 일반화 성능을 향상시킨다.

------

### 문제 12. PCA 목적과 한계

### ✅ 모범답안

> PCA는 데이터의 분산을 최대한 보존하는 방향으로 차원을 축소하는 기법이다.
> 그러나 주성분이 기존 변수의 선형 결합이므로 해석이 어렵다는 한계가 있다.

------

### 문제 13. Bagging vs Boosting

### ✅ 모범답안

> Bagging은 여러 모델을 독립적으로 학습시켜 분산을 감소시키는 기법이다.
> Boosting은 이전 모델의 오분류 데이터를 보완하며 순차적으로 학습한다.
