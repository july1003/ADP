비기봇 해설
다. 테라바이트(Terabyte, TB)
: 1,0244 Byte로, 주로 PC 하드디스크나 SSD의 저장 용량을 나타낼 때 사용됩니다. 테라바이트는 기가바이트(GB)의 1,024배로, 개인용 데이터 저장 장치에서 자주 볼 수 있는 단위입니다.

나. 페타바이트 (Petabyte, PB)
: 1,0245 Byte로, 약 1,000 테라바이트(TB)에 해당하는 데이터 크기 단위입니다. 페타바이트는 대규모 데이터 저장소에서 사용되며, 기업의 데이터 아카이브, 대형 연구 프로젝트, 그리고 영상 데이터 저장 등에 널리 활용됩니다. 예를 들어, 영화 스트리밍 서비스에서 수천 개의 고화질(HD) 영화 파일을 저장하는 데 필요한 용량을 나타낼 때 자주 사용됩니다.

가. 엑사바이트 (Exabyte, EB)
: 1,0246 Byte로, 대규모 데이터 센터의 총 데이터 용량이나 기업 전체의 데이터 저장 규모를 나타낼 때 사용됩니다. 또한, 클라우드 저장 용량, 빅데이터 분석, 그리고 글로벌 인터넷 트래픽 측정을 위한 단위로 자주 사용됩니다.

라. 제타바이트 (Zettabyte, ZB)
: 1,0247 Byte로, 전 세계의 인터넷 트래픽, 클라우드 데이터 저장 용량, 데이터 분석 및 IoT 시대의 초대규모 데이터를 표현하는 데 사용됩니다. 2020년 기준 전 세계 클라우드 저장 용량이 약 40제타바이트(ZB)에 도달한 것으로 추정되며, 이는 전 세계적으로 엄청난 데이터의 증가를 보여줍니다. 제타바이트는 엑사바이트(EB)의 1,024배로, 대규모 데이터 환경에서 점점 더 일반화되고 있는 단위입니다.

1 개인정보 사용자의 책임

비기봇 해설
빅데이터 활용으로 인한 사생활 침해 문제를 해결하기 위해서는 개인정보 제공자의 단순 동의를 넘어, 개인정보를 사용하는 주체가 책임을 지도록 하는 방식이 필요합니다.

1. 개인정보 사용자의 책임
: 개인정보를 사용하는 주체가 책임을 지는 방식은 사생활 침해 문제를 해결하는 데 효과적입니다. 개인정보 제공자의 동의 과정을 간소화하고, 데이터를 활용하는 주체에게 법적·윤리적 책임을 부여하면 개인정보 유출과 악용 가능성을 줄일 수 있습니다.

2. 개인정보 제공자의 동의
: 개인정보 제공자의 동의는 사생활 보호를 위한 기본적인 방식이지만, 반복적이고 형식적인 동의 과정은 비효율적입니다. 또한, 단순 동의만으로는 데이터를 악용하는 문제를 충분히 막을 수 없어 실효성이 낮습니다.

3. 알고리즘 접근 허용
: 알고리즘에 대한 접근권을 제공하는 방안은 데이터 오용 문제를 해결하는 데 초점을 둡니다. 알고리즘의 투명성을 확보하고 부당한 결과에 대해 반증할 수 있는 기회를 제공하는 방식이지만, 이는 데이터 오용과 관련된 문제를 해결하기 위한 방안으로, 사생활 침해 사례와는 직접적인 연관이 없습니다.

4. 결과 기반 책임원칙 고수
: 결과 기반 책임원칙은 예측 알고리즘으로 인해 발생하는 피해를 최소화하고, 책임의 기준을 명확히 하는 데 목적이 있습니다. 이는 책임원칙 훼손 문제를 해결하기 위한 방안으로, 사생활 침해와는 관련이 없습니다. 따라서 본 사례에서는 적합하지 않은 선택지입니다.

비기봇 해설
빅데이터는 데이터 수집 비용 감소와 기술의 발전으로 인해 표본조사 방식에서 전수조사 방식으로 전환되었습니다. 이는 특정 그룹의 편중된 의견이나 데이터 왜곡을 줄이고 더 다양한 정보를 기반으로 객관적이고 포괄적인 결과를 도출하는 데 기여합니다.

1. 전수조사로의 변화
: 전수조사는 표본조사를 넘어 전체 데이터를 수집하여 분석하는 방식으로, 특정 그룹의 의견 편중 문제를 효과적으로 해결한다. 전수조사는 방대한 데이터를 활용해 샘플링이 주지 못하는 숨은 패턴과 정보를 발견할 수 있어, 왜곡된 결과를 줄이고 보다 신뢰할 수 있는 결과를 도출하는 데 기여한다.

2. 사후처리로의 변화
: 사후처리는 데이터 수집 이후 모든 데이터를 저장하고 다양한 방식으로 처리하는 것을 의미한다. 이는 데이터의 처리 방식 변화와 관련이 있을 뿐, 특정 그룹의 편향된 의견 문제와는 직접적인 연관이 없다.

3. 상관관계로의 변화
: 상관관계는 데이터 간의 관계성을 분석하는 방식으로, 특정 데이터를 기반으로 패턴을 발견할 수 있다. 그러나 상관관계로의 변화는 특정 그룹의 편향된 의견 문제를 해결하는 직접적인 방식이라기보다는, 데이터 간 연관성을 탐구하는 데 초점이 맞춰져 있다.

4. 인과관계로의 변화
: 인과관계는 원인과 결과를 밝히는 데 초점을 맞추는 방식이다. 그러나 빅데이터 분석에서는 인과관계보다는 상관관계 중심으로 변화하고 있기 때문에, 특정 그룹의 편중된 의견 문제를 해결하기 위한 주요 변화와는 거리가 있다.

(A)A마트는 130원에, B마트는 200원에 연필을 판매한다.
: 단순히 수집된 사실의 나열로, 맥락이나 의미가 부여되지 않은 상태입니다. 가격 정보(130원, 200원)는 관찰된 사실로, 이를 비교하거나 해석하지 않았기 때문에 데이터로 분류됩니다.

(D)A마트의 다른 상품도 B마트보다 저렴할 것이다.

비기봇 해설
데이터 웨어하우스(Data Warehouse)는 대규모 데이터 분석과 의사결정을 지원하기 위해 설계된 데이터 저장소입니다. 데이터 웨어하우스는 실시간 데이터 갱신보다는 정기적인 데이터 로딩과 분석 적합성을 더 중요하게 생각합니다.

4. HDFS는 대량의 데이터 처리에 최적화되어 있지만, 단일 사용자의 읽기 및 쓰기 작업은 NAS(Network Attached Storage)나 SAN(Storage Area Network)보다 빠르지 않을 수 있습니다.

비기봇 해설
Servitization(서비타이제이션)은 제조업 기업이 단순히 제품을 판매하는 것에서 벗어나, 제품과 서비스를 결합하여 새로운 가치를 창출하는 비즈니스 모델을 의미합니다. 제품을 단순히 판매하는 것이 아니라, 서비스를 추가하여 지속적인 고객 관계를 형성하고 수익을 창출하는 것이 핵심입니다.

1. 제품과 서비스의 결합을 통한 기업의 새로운 비즈니스 모델
: Servitization의 핵심 개념은 제품(Product)과 서비스(Service)의 결합을 통한 새로운 비즈니스 모델 창출입니다. 롤스로이스(Rolls-Royce)는 엔진을 판매하는 것이 아니라 "Power-by-the-Hour" 서비스 모델을 도입하여, 사용한 시간만큼 요금을 부과하는 방식으로 변화했습니다. 단순 제품 판매에서 지속적인 서비스 제공을 통한 비즈니스 모델로 전환하는 것이 Servitization의 핵심입니다.

2. 제품의 서비스화와 서비스의 상품화를 모두 포함하는 결합 비즈니스 모델
: Servitization은 제품을 단순 판매하는 것이 아니라 서비스 형태로 제공하는 "제품의 서비스화"와, 반대로 서비스 자체를 하나의 상품처럼 제공하는 "서비스의 상품화"를 모두 포함하는 개념입니다. 예를 들어, 자동차 공유 서비스(서비스의 상품화)나 SaaS(Software as a Service, 제품의 서비스화) 등이 이에 해당됩니다.

3. 파이프 라인 비즈니스를 넘어 플랫폼 비즈니스를 위한 모델
: 전통적인 제조업(파이프라인 비즈니스)은 제품을 생산하고 판매하는 방식이지만, Servitization은 플랫폼을 활용하여 지속적인 서비스를 제공하는 방식으로 확장될 수 있습니다. 예를 들어, 제조업체가 단순히 기계를 판매하는 것이 아니라, IoT 기반으로 기계를 모니터링하고 예측 유지보수를 제공하는 서비스형 모델을 운영하는 방식이 이에 해당합니다.

4. 생산된 제품의 특화를 통한 경쟁우위 확보 모델
: Servitization은 제품 자체를 특화하는 것이 아니라, 제품과 서비스를 결합하여 부가가치를 창출하는 것이 핵심입니다. 단순히 제품의 성능이나 품질을 향상시키는 것만으로는 Servitization의 개념과 맞지 않습니다. 예를 들어, 고급 기능이 추가된 스마트폰을 출시하는 것은 제품 특화 전략이지만, 스마트폰을 기반으로 한 구독형 콘텐츠 서비스나 애프터서비스 패키지를 제공하는 것은 Servitization에 해당합니다.

비기봇 해설
분석 마스터 플랜 수립에서 과제의 우선순위를 평가할 때, 일반적으로 시급성과 난이도를 고려하지만, 난이도가 높은 과제가 반드시 우선순위가 높은 것은 아닙니다.

1. 우선순위 선정 매트릭스에서 시급성과 난이도가 높은 과제가 우선순위가 높다.
: 우선순위 선정 매트릭스에서 시급성이 높더라도 난이도가 높으면 실행이 어렵기 때문에 우선순위가 반드시 높은 것은 아닙니다. 일반적으로 시급성이 높고 난이도가 낮은 과제가 먼저 수행되며, 난이도가 높은 과제는 추가적인 투자 및 준비가 필요할 수 있습니다. 따라서 난이도가 높다고 해서 반드시 우선순위가 높은 것은 아닙니다.

2. 우선순위 선정 매트릭스에서 난이도와 시급성으로 구분한다.
: 우선순위를 정할 때 일반적으로 시급성과 난이도를 기준으로 매트릭스를 구성합니다. 시급성이 높은 과제는 즉각적인 비즈니스 효과를 기대할 수 있는 반면, 난이도가 높은 과제는 구현에 많은 시간과 비용이 필요할 수 있습니다.

3. 시급성과 전략적 중요도는 Value인 비즈니스 효과에 대한 구분이다.
: 시급성과 전략적 중요도는 기업이 분석 프로젝트를 수행할 때 비즈니스적으로 가져올 가치(Value)를 평가하는 기준이 됩니다. 즉, 어떤 과제가 기업의 목표 달성에 중요한 영향을 미치는지를 평가하는 요소입니다.

4. 난이도는 Volume, Variety, Velocity의 투자비용 요소에 대한 구분이다.
: 난이도는 데이터의 특성(3V: Volume, Variety, Velocity)에 의해 결정됩니다.
• Volume(데이터 양): 처리해야 할 데이터의 크기
• Variety(다양성): 데이터의 유형(정형, 비정형, 반정형)
• Velocity(속도): 데이터의 생성 및 처리 속도
이러한 요소들이 난이도와 직결되며, 난이도가 높을수록 필요한 기술적 역량과 비용이 증가합니다.

2 업무별 적합한 분석기법

비기봇 해설
분석 준비도는 기업이 데이터 분석을 수행할 수 있는 역량을 평가하는 기준이며, 이 중 분석 업무 파악 영역은 기업이 수행하는 분석의 종류를 분류하는 과정입니다.

1. 최적화 분석 업무
: 최적화 분석(Prescriptive Analytics)은 가장 발전된 분석 형태로, 예측된 결과를 기반으로 최적의 의사결정을 지원하는 분석입니다. 예를 들어, 물류 최적화, 가격 최적화, 생산 계획 최적화 등의 분석이 이에 해당합니다.

2. 업무별 적합한 분석기법
: 분석 업무 파악 영역은 기업이 수행하는 분석 유형을 분류하는 것이지, 해당 업무에 적합한 분석 기법을 선정하는 과정이 아닙니다. 예를 들어, 기업이 예측 분석을 수행할 때 시계열 분석, 머신러닝 등 어떤 기법을 적용할지를 결정하는 과정은 "분석 기법" 영역에 해당합니다.

3. 예측 분석 업무
: 예측 분석(Predictive Analytics)은 과거 데이터를 바탕으로 미래를 예측하는 분석 방법입니다. 머신러닝, 시계열 분석, 회귀 분석 등을 활용하여 향후 발생할 가능성이 높은 이벤트를 예측하는 데 사용됩니다.

4. 발생한 사실 분석 업무
: 발생한 사실 분석(Descriptive Analytics)은 과거 데이터를 기반으로 현재 상태를 설명하는 분석입니다. 예를 들어, 매출 데이터 분석, 고객 구매 패턴 분석 등이 이에 해당합니다. 이는 데이터 분석의 가장 기본적인 단계입니다.

비기봇 해설
STEEP 분석은 비즈니스 환경을 분석하는 방법 중 하나로, 사회(Social), 기술(Technological), 경제(Economic), 환경(Environmental), 정치(Political) 다섯 가지 요소를 고려하여 분석 기회를 탐색하는 데 사용됩니다.

1. 환경적(Environment) 정책
: 환경적 요소는 기후 변화, 지속 가능성, 친환경 정책 등의 변화를 포함합니다. 예를 들어, 탄소 배출 규제에 따른 친환경 제품 개발 기회는 환경적 요소에 해당합니다. 하지만 문제에서 제시한 고객 확장과 문화적 트렌드 변화와는 직접적인 관련이 없습니다.

2. 경제적(Economic) 정책
: 경제적 요소는 시장의 경제 성장, 소비자 구매력, 인플레이션, 금융 정책 등의 요소를 포함합니다. 예를 들어, 경기 침체로 인한 저가형 제품 시장 확대는 경제적 요소에 해당합니다. 하지만 고객 정책 변화 및 문화적·구조적 트렌드 변화를 중심으로 한 분석 기회와는 거리가 있습니다.

3. 사회적(Social) 정책
: 사회적 요소는 인구 변화, 소비자 행동, 문화적 트렌드, 사회적 가치관 변화 등을 포함합니다. 문제에서 언급된 "현재 고객을 확장하여 전체 시장을 대상으로"라는 표현은 소비자 행동 및 문화적 변화와 관련된 내용으로, 사회적 요소에 해당합니다. 예를 들어, 비즈니스 모델이 MZ세대를 타겟으로 확장되거나, ESG(환경·사회·거버넌스) 가치가 중요해지는 트렌드를 반영하는 분석 기회는 사회적 요소에서 비롯됩니다.

4. 정치적(Political) 정책
: 정치적 요소는 정부 규제, 법률, 공공정책, 세금 정책 등의 변화를 포함합니다. 예를 들어, 데이터 보호법 강화로 인한 개인정보 처리 방식의 변화는 정치적 요소에 해당합니다. 하지만 문제에서 제시된 "문화적, 구조적 트렌드 변화"는 정치적 요소보다는 사회적 요소에 더 가깝습니다.

1 단일 연결법

비기봇 해설
1. 단일 연결법 (Single Linkage)은 두 군집 간의 거리(혹은 유사성)는 두 군집 내의 모든 점 간의 거리 중 최소값을 사용합니다. 이 방법은 한 군집의 점이 다른 군집의 점과 가까운 경우, 두 군집이 병합되도록 합니다. 밀집된 군집을 잘 찾고, 군집 간의 연결을 잘 형성하나 "사슬 효과"로 인해 군집이 길게 늘어지는 경향이 있습니다.

2. 와드 연결법 (Ward's Linkage)은 두 군집 간의 거리를 두 군집을 병합했을 때의 분산 증가량으로 정의합니다. 즉, 두 군집을 합쳤을 때의 총 제곱합의 증가량을 최소화하는 군집 간의 병합을 선택합니다. 이 방법은 군집 내의 점들이 서로 가까운 경우에 효과적이며, 상대적으로 균형 잡힌 군집을 형성합니다. 군집의 크기와 모양에 대해 유연하게 대응할 수 있으나 계산이 복잡하고 대규모 데이터에 대해 느릴 수 있습니다.

3. 완전 연결법(Complete Linkage)는 두 군집 간의 거리를 두 군집 내의 모든 점 간의 거리 중 최대값으로 정의합니다. 이 방법은 밀집된 군집을 형성하는 데 효과적이며, 군집 간의 연결을 강하게 만듭니다. 군집의 경계가 뚜렷하고, 긴 군집을 피하는 경향이 있으나, 군집의 형태에 따라 비효율적일 수 있습니다.

4. 평균 연결법 (Average Linkage)은 두 군집 간의 거리를 두 군집 내의 모든 점 간의 거리의 평균값으로 정의합니다. 이 방법은 각 군집의 모든 쌍의 점 간의 거리를 고려하여 군집 간의 거리를 정의합니다. 균형 잡힌 군집을 형성하며, 군집의 모양에 대한 영향을 덜 받으나 계산량이 많아질 수 있습니다.

- 전체 고객 중 신용이 불량한 사람이 10% 임
- 신용평가 모형을 통해 예측한 결과 전체 고객 중 20%가 신용이 불량하다고 진단 됨
- 신용이 불량할 것으로 예측한 사람의 45%가 실제로 신용이 불량한 것으로 확인 됨
  1. 0.9

  4. 0.2

비기봇 해설
정답은 1번입니다.

이 문제는 민감도(Sensitivity) 또는 재현율(Recall)을 구하는 문제입니다.

[주어진 정보]

• 실제 신용불량자 비율: 10%
• 모델이 신용불량으로 예측한 비율: 20%
• 신용불량으로 예측한 사람 중 실제로 신용불량인 비율(정밀도): 45%

[단계별 해결과정]

▶ 1단계: 전체 고객 수를 1000명으로 가정

실제 신용불량자: 1000 × 0.1 = 100명
실제 신용양호자: 1000 × 0.9 = 900명

▶ 2단계: 예측 결과 분석

신용불량으로 예측된 사람: 1000 × 0.2 = 200명
이 중 실제로 신용불량인 사람: 200 × 0.45 = 90명

▶ 3단계: 혼동행렬(Confusion Matrix) 구성

• True Positive (TP): 실제 불량 → 불량 예측 = 90명
• False Negative (FN): 실제 불량 → 양호 예측 = 100 - 90 = 10명
• False Positive (FP): 실제 양호 → 불량 예측 = 200 - 90 = 110명
• True Negative (TN): 실제 양호 → 양호 예측 = 900 - 110 = 790명

▶ 4단계: 민감도 계산
민감도 = TP / (TP + FN) = 90 / (90 + 10) = 90/100 = 0.9
답: 90%

실제로 신용이 불량한 사람을 신용이 불량하다고 예측할 확률은 90%입니다. 이는 실제 신용불량자 100명 중 90명을 올바르게 신용불량으로 예측했다는 의미입니다.
이 지표는 신용평가에서 매우 중요한데, 실제 신용불량자를 놓치는 것(False Negative)은 은행에 큰 손실을 가져올 수 있기 때문입니다.

따라서 정답은 '1. 0.9'입니다.

60. 데이터 분할에 대한 설명으로 가장 적절하지 않은 것은?
  1. 교차검증은 모델의 일반화 성능을 평가하기 위해 데이터를 여러 번 나누어 검증하는 방법이다.

비기봇 해설
1. 맞는 설명입니다. 교차검증(Cross-Validation)은 주어진 데이터셋을 여러 번 나누어 모델의 성능을 평가하는 기법입니다. 가장 일반적인 형태인 k-겹 교차검증(k-fold cross-validation)에서는 데이터를 k개의 부분으로 나누고, 각 부분을 한 번씩 검증용 데이터로 사용하며 나머지 k-1 부분을 훈련 데이터로 사용합니다. 이렇게 하면 모델의 일반화 성능을 보다 정확하게 평가할 수 있으며, 데이터의 특정 분할에 의한 편향을 줄일 수 있습니다.

2. 맞는 설명입니다. LOOCV는 특별한 형태의 교차검증으로, 데이터셋의 각 데이터 포인트를 한 번씩 검증용 데이터로 사용하고 나머지 모든 데이터 포인트를 훈련 데이터로 사용하는 방식입니다. 예를 들어, 데이터셋에 100개의 데이터 포인트가 있다면, LOOCV는 100번의 훈련과 검증을 수행하게 됩니다. 이 방법은 데이터셋이 작을 때 유용하지만, 계산 비용이 매우 높아질 수 있는 단점이 있습니다.

3. 이 설명은 적절하지 않습니다. 홀드 아웃(Hold-Out) 방법은 데이터셋을 간편하게 두 개의 부분(훈련 데이터와 검증 데이터)으로 나누는 기법입니다. 일반적으로 70% 또는 80%의 데이터를 훈련에 사용하고 나머지를 검증에 사용합니다. 그러나 이 방법은 특히 데이터셋이 작은 경우, 일반화 성능을 보장하지 않습니다. 홀드 아웃 방식은 특정한 데이터 분할에 의존하기 때문에, 데이터의 특성에 따라 모델의 성능이 크게 달라질 수 있습니다. 따라서, 홀드 아웃 방식이 가장 많이 사용된다고 할 수는 없습니다.

4. 맞는 설명입니다. 데이터 분할 없이 모델을 훈련하면, 모델이 훈련 데이터에 과도하게 적합하게 되어 새로운 데이터에 대한 예측 성능이 저하될 수 있습니다. 과적합은 모델이 훈련 데이터의 노이즈나 특이점을 학습하게 되는 현상으로, 일반화 성능을 떨어뜨립니다. 데이터 분할은 이러한 과적합을 방지하는 중요한 방법 중 하나입니다.
