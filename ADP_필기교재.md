
# ADP 필기시험 교재 노트 
# 1. 데이터 이해

## 1.1 데이터의 이해

### 1.1.1 데이터와 정보

#### ① 개념 정의
데이터(Data)는 관찰이나 측정을 통해 수집된 **가공되지 않은 사실의 집합**을 의미한다.  
정보(Information)는 데이터를 특정 목적에 맞게 처리·해석하여 **의사결정에 활용 가능한 형태**로 만든 것이다.

#### ② 배경 및 필요성
기업과 조직은 방대한 데이터를 보유하고 있으나, 데이터 자체는 가치가 없다.  
의사결정에 활용될 수 있을 때 비로소 정보가 되며, 이 과정이 데이터 분석의 출발점이다.

#### ③ 핵심 구분 포인트
- 데이터: 사실, 원천, 객관적
- 정보: 맥락 포함, 의미 부여, 목적 지향적
- 동일한 데이터라도 목적에 따라 정보의 가치가 달라진다.

#### ④ 시험 출제 포인트
- “데이터와 정보의 차이를 설명하시오” (서술형)
- ‘데이터 = 정보’로 혼동하는 선지 출제

#### ⑤ 시험용 핵심 문장
> 데이터는 가공되지 않은 사실이며, 정보는 의사결정에 활용 가능한 데이터이다.

---

### 1.1.2 데이터베이스의 정의와 특징

#### ① 개념 정의
데이터베이스(Database)는 조직의 데이터를 **통합하여 저장하고, 공동으로 활용**할 수 있도록 관리하는 데이터 저장소이다.

#### ② 데이터베이스의 필요성
- 데이터 중복 최소화
- 데이터 무결성 및 일관성 확보
- 다수 사용자 동시 접근 지원

#### ③ 주요 특징
- 통합성: 중복 최소화
- 일관성: 동일 데이터의 동일 값 유지
- 무결성: 제약조건을 통한 정확성 보장
- 보안성: 권한 기반 접근 제어

#### ④ OLTP와 DW 비교
- OLTP: 트랜잭션 처리, 정규화 구조
- DW: 분석·집계 목적, 비정규화 구조

#### ⑤ 시험 출제 포인트
- DB 특징 중 틀린 것 고르기
- OLTP와 DW 혼동 유도

---

### 1.1.3 데이터베이스 활용

#### ① 활용 목적
- 운영 시스템 지원
- 분석 및 의사결정 지원
- 이력 관리 및 리포팅

#### ② 활용 유형
- 운영 DB: 실시간 처리
- DW/DM: 분석 중심
- ODS: 운영과 분석의 중간 단계

#### ③ 시험 출제 포인트
- “ODS의 역할”과 연계 출제

## 1.2 데이터의 가치와 미래

### 1.2.1 빅데이터의 이해

#### ① 정의
빅데이터(Big Data)는 기존 데이터베이스 기술로 처리하기 어려운
대용량·다양성·고속성 데이터를 의미한다.

#### ② 5V 특성
- Volume: 대용량
- Velocity: 생성·처리 속도
- Variety: 다양한 형태
- Veracity: 신뢰성
- Value: 가치 창출

#### ③ 시험 출제 포인트
- 5V 중 Value 강조 여부
- 단순 크기 개념으로 오해 유도

---

### 1.2.2 빅데이터의 가치와 영향

#### ① 조직 측면 가치
- 데이터 기반 의사결정
- 비용 절감
- 고객 맞춤 서비스

#### ② 산업적 영향
- 금융, 제조, 유통, 공공 전반 확산

#### ③ 시험 포인트
- “빅데이터의 가치는 기술이 아니라 활용에 있다”

---

### 1.2.3 비즈니스 모델

#### ① 데이터 기반 비즈니스 모델
- 데이터 수집 → 분석 → 가치 창출

#### ② 주요 유형
- 플랫폼 모델
- 구독 모델
- 광고 기반 모델
- 데이터 판매 모델

#### ③ 시험 포인트
- ‘분석 자체’가 아니라 ‘수익 구조’ 관점

---

### 1.2.4 위기 요인과 통제 방안

#### ① 주요 위기 요인
- 개인정보 침해
- 데이터 편향
- 보안 위협

#### ② 통제 방안
- 데이터 거버넌스
- 익명화/가명화
- 접근 권한 관리

#### ③ 시험 포인트
- 윤리·보안 문제 단골 출제

---

### 1.2.5 미래의 빅데이터

- 실시간 분석 확대
- AI·AutoML 결합
- MLOps 기반 운영 자동화

---

## 1.3 가치 창출을 위한 데이터 사이언스와 전략 인사이트

### 1.3.1 빅데이터 분석과 전략 인사이트

### 1.3.3 빅데이터와 데이터 사이언스의 미래
- 책임 있는 AI
- 자동화 분석

# 4. 데이터분석

## 4.1 R 기초와 데이터 마트

### 4.1.1 R 기초

R은 통계 분석과 데이터 분석을 위해 설계된 프로그래밍 언어이자 환경이다.  
ADP 필기에서는 **문법 자체보다 분석 도구로서의 역할 이해**가 핵심이다.

- 데이터 구조: vector, matrix, data.frame, list
- 분석 흐름: 데이터 불러오기 → 전처리 → 분석 → 시각화
- 시험 포인트: R은 분석 도구이지 DB 대체 수단이 아님

---

### 4.1.2 데이터 마트(Data Mart)

데이터 마트는 데이터 웨어하우스에서 특정 부서나 주제에 맞게 구성된 분석용 데이터 저장소이다.

- 목적: 사용자 친화적 분석
- 구조: 스타 스키마 중심
- 특징: 요약 데이터, 조회 성능 중시

시험 포인트  

> DM은 DW를 대체하지 않으며, DW 기반 하위 구조이다.

---

### 4.1.3 결측값 처리와 이상값 검색

**결측값 처리 방법**

- 삭제: 정보 손실 위험
- 대체: 평균, 중앙값, 회귀 기반
- 고급: 다중 대치법

**이상값 탐지**

- 통계적 방법: Z-score, IQR
- 모델 기반: Isolation Forest

시험 포인트  

- 무조건 제거 ❌  
- 분석 목적에 따라 판단 ⭕

---

## 4.2 통계분석

### 4.2.1 통계학 개론

통계학은 불확실한 데이터를 기반으로 **의사결정을 지원하는 학문**이다.

- 기술통계: 데이터 요약
- 추론통계: 모집단 추정

---

### 4.2.2 기초 통계분석

**주요 지표**

- 평균, 분산, 표준편차
- 왜도, 첨도

**가설검정**

- 귀무가설 / 대립가설
- p-value 정의 (매우 중요)

시험 포인트  

> p-value는 귀무가설이 참이라는 가정 하에서 관측값 이상이 나올 확률이다.

---

### 4.2.3 다변량 분석

- 회귀분석: 종속·독립 변수 관계
- PCA: 차원 축소, 해석력 한계
- 요인분석: 잠재 요인 도출

시험 포인트  

- PCA = 변수 선택 ❌  
- PCA = 축 생성 ⭕

---

### 4.2.4 시계열 예측

- 구성 요소: 추세, 계절성, 불규칙
- ARIMA 모델 개요

시험 포인트  

- 과거 패턴 기반 예측
- 외삽 한계 인식

---

시험 포인트

- Support / Confidence / Lift

시험 포인트
