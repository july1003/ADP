1. 데이터는 그 형태에 따라 정성 데이터와 정량 데이터로 구분된다. 다음 중 정성 데이터에 속하는 것은?
  1. 풍향
  2. 습도
  3. 기상특보
  4. 1시간 강수량

3 기상특보

해설
정량적 데이터의 형태는 수치, 도형, 기호 등으로 기술이 되며, 정성 데이터의 형태는 언어, 문자 등으로 기술된다.
비기봇 해설
은 3번입니다.

정성 데이터는 수치로 표현되지 않고, 주로 질적 특성을 나타내는 데이터입니다. 반면, 정량 데이터는 수치로 표현되어 양적 특성을 나타냅니다.

1. 풍향
: 풍향은 바람이 불어오는 방향을 나타내며, 주로 동서남북과 같은 방향으로 표현됩니다. 이는 수치보다는 방향이라는 질적 특성을 나타내므로 정성 데이터에 해당합니다. 그러나, 풍향을 각도로 표현할 경우 정량 데이터로 볼 수 있습니다.

2. 습도
: 습도는 공기 중 수분의 양을 백분율로 나타내는 수치 데이터입니다. 이는 정량 데이터에 해당합니다.

3. 기상특보
: 기상특보는 특정 기상 현상에 대한 경고나 주의보를 의미하며, 이는 수치보다는 상황이나 상태를 설명하는 질적 특성을 나타내므로 정성 데이터에 해당합니다.

4. 1시간 강수량
: 1시간 강수량은 특정 시간 동안 내린 비의 양을 밀리미터(mm)로 측정한 수치 데이터입니다. 이는 정량 데이터에 해당합니다.


기상특보는 특정 기상 현상에 대한 경고나 주의보로, 수치로 표현되지 않고 상황을 설명하는 질적 특성을 가지고 있어 정성 데이터에 해당합니다. 다른 선택지들은 모두 수치로 표현되는 정량 데이터입니다.

4 추상화

해설

비기봇 해설
1. 공통화: 암묵지를 공유하고 확산하는 과정으로, 개인의 암묵지를 다른 사람들과 공유하여 공통의 암묵지를 만드는 것을 의미한다. 이는 암묵지와 형식지의 상호작용과 관련이 있다.

2. 내면화: 형식지를 개인의 암묵지로 만드는 과정으로, 형식지를 통해 새로운 지식을 습득하고 이를 자신의 경험과 연결시켜 내면화하는 것을 의미한다. 이는 암묵지와 형식지의 상호작용과 관련이 있다.

3. 연결화: 암묵지를 형식지로 만드는 과정으로, 개인의 암묵지를 언어나 기호 등으로 표현하여 형식지로 만드는 것을 의미한다. 이는 암묵지와 형식지의 상호작용과 관련이 있다.

4. 추상화: 구체적인 사실이나 개념을 단순화시켜 일반적인 원리나 개념으로 만드는 과정을 의미한다. 이는 암묵지와 형식지의 상호작용과는 직접적인 관련이 없다.

SQL의 집계 함수는 데이터베이스에서 데이터를 요약하고 분석하는 데 사용됩니다. 이 문제는 다양한 데이터 타입에 적용할 수 있는 집계 함수를 찾는 것입니다.

1. AVG
: AVG 함수는 평균을 계산하는 함수로, 주로 숫자 데이터 타입에 사용됩니다. 문자열이나 다른 데이터 타입에는 사용할 수 없습니다.

2. COUNT
: COUNT 함수는 데이터의 개수를 세는 함수로, 데이터의 타입에 상관없이 모든 데이터 타입에 사용할 수 있습니다. NULL 값을 제외하고 개수를 셉니다.

3. SUM
: SUM 함수는 합계를 계산하는 함수로, 숫자 데이터 타입에만 사용 가능합니다. 문자열이나 다른 데이터 타입에는 사용할 수 없습니다.

4. STDDEV
: STDDEV 함수는 표준 편차를 계산하는 함수로, 주로 숫자 데이터 타입에 사용됩니다. 문자열이나 다른 데이터 타입에는 사용할 수 없습니다.


COUNT 함수는 데이터의 개수를 세는 데 사용되며, 데이터의 타입에 상관없이 모든 데이터 타입에 적용할 수 있습니다. 다른 함수들은 주로 숫자 데이터 타입에만 사용 가능하므로, COUNT가 입니다.

개인정보 비식별화 기법은 개인을 식별할 수 없도록 데이터를 처리하는 방법입니다. 이러한 기법은 데이터의 프라이버시를 보호하면서도 데이터를 활용할 수 있도록 돕습니다.

1. 총계처리 – 데이터의 총합 값을 보임으로써 개별 데이터의 값을 보이지 않도록 함
: 총계처리는 데이터의 총합을 보여줌으로써 개별 데이터의 값을 숨기는 방법입니다. 이는 데이터의 개별 항목을 식별할 수 없게 만듭니다.

2. 데이터 마스킹 – 개인 식별에 중요한 데이터 값을 삭제
: 데이터 마스킹은 개인 식별에 중요한 데이터를 삭제하는 것이 아니라, 데이터를 변형하여 원래의 값을 숨기는 방법입니다. 따라서 설명이 부정확합니다.

3. 가명처리 - 개인 식별에 중요한 데이터를 식별 할 수 없는 다른 값으로 변경
: 가명처리는 개인 식별 정보를 다른 값으로 대체하여 식별할 수 없도록 하는 방법입니다. 이는 비식별화의 대표적인 방법 중 하나입니다.

4. 범주화 – 데이터의 값을 범주의 값으로 변환하여 값을 감춤
: 범주화는 데이터를 특정 범주로 변환하여 개별 값을 숨기는 방법입니다. 이는 데이터의 세부 정보를 감추는 데 효과적입니다.

데이터를 가공, 처리하여 얻을 수 있는 것은 데이터(Data), 정보(Information), 지식(Knowledge), 지혜(Wisdom)이다. 또한 이들은 계층적 구조로서 DIKW 피라미드를 형성한다.
비기봇 해설
1. 지혜: 데이터를 가공 및 처리하여 얻은 정보를 바탕으로 경험과 판단력을 통해 얻는 것이 지혜입니다. 따라서 데이터를 통해 지혜를 얻을 수 있습니다.

2. 정보: 데이터를 가공 및 처리하면 정보를 얻을 수 있습니다. 정보는 데이터가 특정 문맥에서 해석되어 의미를 가지게 된 것입니다.

3. 기호: 기호는 특정한 의미를 나타내는 데 사용되는 표시나 부호입니다. 데이터를 가공 및 처리하여 기호를 얻는 것은 부적절합니다. 기호는 이미 의미를 가지고 있으며, 데이터의 가공 및 처리와는 별개의 개념입니다.

4. 지식: 데이터를 가공 및 처리하여 얻은 정보를 이해하고 해석하는 과정을 통해 지식을 얻을 수 있습니다. 따라서 데이터를 통해 지식을 얻을 수 있습니다.

비기봇 해설
은 2번입니다.

일차원적 분석은 특정 산업에서 주로 사용되는 분석 방법으로, 주로 단순한 데이터 패턴을 파악하거나 기본적인 예측을 수행하는 데 사용됩니다. 예를 들어, 에너지 산업에서는 트레이딩, 공급 및 수요 예측과 같은 일차원적 분석이 중요합니다. 이는 에너지의 생산과 소비를 예측하고, 시장에서의 거래를 최적화하는 데 필수적입니다. 이러한 분석은 복잡한 전략보다는 기본적인 데이터 흐름과 패턴을 이해하는 데 중점을 둡니다.

1. 소매업
: 소매업에서는 주로 판촉, 매대 관리, 수요 예측, 재고 보충과 같은 분석이 이루어집니다. 이는 고객의 구매 패턴을 이해하고, 적절한 시기에 적절한 상품을 제공하기 위한 것입니다. 그러나 트레이딩과 같은 활동은 소매업의 주요 분석 활동이 아닙니다.

2. 에너지
: 에너지 산업에서는 트레이딩, 공급 및 수요 예측이 핵심 분석 활동입니다. 이는 에너지의 생산과 소비를 예측하고, 시장에서의 거래를 최적화하는 데 필수적입니다. 따라서 문제에서 제시된 일차원적 분석 사례는 에너지 산업에 적합합니다.

3. 운송업
: 운송업에서는 일정 관리, 노선 배정, 수익 관리와 같은 분석이 주로 이루어집니다. 이는 효율적인 운송 경로를 계획하고, 수익성을 극대화하기 위한 것입니다. 트레이딩과 같은 활동은 운송업의 주요 분석 활동이 아닙니다.

4. 금융서비스
: 금융서비스에서는 신용점수 산정, 사기 탐지, 가격 책정, 프로그램 트레이딩과 같은 복잡한 분석이 이루어집니다. 이는 금융 시장의 복잡한 데이터 패턴을 이해하고, 리스크를 관리하기 위한 것입니다. 문제에서 제시된 일차원적 분석 사례는 금융서비스와는 다소 거리가 있습니다.

문제에서 제시된 트레이딩, 공급, 수요 예측은 에너지 산업에서 주로 사용되는 일차원적 분석 방법입니다. 에너지 산업은 이러한 분석을 통해 에너지의 생산과 소비를 예측하고, 시장에서의 거래를 최적화합니다. 다른 선지들은 각각의 산업에서 주로 사용되는 분석 방법과 일치하지 않기 때문에 입니다.
따라서, 은 '2. 에너지' 입니다.

고객관계관리(CRM)는 기업이 고객과의 관계를 관리하고 강화하기 위해 사용하는 시스템입니다. CRM은 고객의 데이터를 수집하고 분석하여 고객의 요구와 행동을 이해하고, 이를 통해 고객 만족도를 높이고 기업의 수익성을 향상시키는 것을 목표로 합니다.

4. 단순한 정보의 수집에서 탈피, 분석 중심의 시스템 구축 지향
: CRM은 단순한 정보 수집을 넘어 고객 데이터를 분석하여 고객과의 관계를 강화하는 시스템을 지향합니다. 이는 CRM의 핵심 기능 중 하나입니다.

1. ERP
: ERP는 Enterprise Resource Planning의 약자로, 기업의 모든 자원을 통합적으로 관리하여 경영 효율성을 높이는 시스템입니다. 생산, 재무, 인사, 물류 등 다양한 부문을 하나의 시스템으로 통합하여 운영합니다.

2. CRM
: CRM은 Customer Relationship Management의 약자로, 고객과의 관계를 관리하고 강화하는 데 중점을 둔 시스템입니다. 주로 고객 데이터 분석을 통해 마케팅 전략을 수립하는 데 사용됩니다.

3. SCM
: SCM은 Supply Chain Management의 약자로, 공급망 전체를 관리하여 효율성을 높이는 시스템입니다. 주로 물류와 관련된 프로세스를 최적화하는 데 중점을 둡니다.

4. KMS
: KMS는 Knowledge Management System의 약자로, 조직 내 지식을 체계적으로 관리하고 공유하는 시스템입니다. 주로 지식의 생성, 저장, 공유를 통해 조직의 학습 능력을 향상시키는 데 사용됩니다.
